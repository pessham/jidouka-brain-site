Playwrightとは

Playwrightは、Microsoftが2020年1月に公開したオープンソースのブラウザテスト自動化ツールです。 ページを開いた直後に急いで「次へ」を押して、「あれ？反応しない」となることがありますよね。 Playwrightは、ボタンや入力欄が操作可能になるまで自動で待ってくれるので、そういうタイミングのズレを減らしやすいです。

専門用語では、同じコードなのに成功したり失敗したりする不安定な状態をFlaky Testと言います。 この不安定さを減らせるので、ブラウザ操作の自動化に向いています。

まず何ができるか

自動化できる作業の見取り図

メール・請求書

メールの問い合わせを読んでスプレッドシートへ転記。請求書メールだけ拾ってPDFをDriveへ保存。

予定・タスク

カレンダー予定を見て前日にリマインド。商談メモから次回タスクをGoogleカレンダーへ登録。

問い合わせ・営業管理

届いた問い合わせを整理し、対応状況や商談メモを自社の管理シートへ記録。外部フォームへの営業自動送信は推奨しません。

コンテンツ制作

note下書き、X投稿案、YouTube概要欄、音声配信の原稿化などを半自動化。

EC・予約サイト

商品名、価格、在庫数、説明文、配送条件、公開ステータスを管理画面へ入力。

第1章

3つの道具を使い分ける

Playwright MCP

AIに実際のブラウザを見せながら、画面確認や簡単な操作を手伝ってもらうための道具です。

Playwrightコード

毎日繰り返す作業を、ボタン1つで動くロボットにする本命です。.command化すればダブルクリックで動かせます。

Computer Use

人間のPC操作に近い形で動かす方法です。便利ですが、まずはMCPとコードから考えるのがおすすめです。

第2章

4フェーズの型

  1. Phase 1　業務を録画する

    MacはShift + Command + 5、WindowsはWindowsキー + G。まず自分の作業をそのまま録ります。

  2. Phase 2　最小版を作る

    いきなり全部ではなく、ログインして投稿画面まで行く、タイトルだけ入れる、のように小さく作ります。

  3. Phase 3　動かして直す

    止まったら、エラー文、スクショ、URL、本来どうなってほしかったかをAIに渡します。

  4. Phase 4　戻れる設計にする

    ブラウザを閉じない、ログを残す、途中から再開できる。ここまで入ると毎日使える道具に近づきます。

  5. Phase 5　応用編

    固定、条件分岐、AI判断の3段階で融通を持たせます。最初の1個目では無理にやらなくて大丈夫です。

第3章

パターン辞典は、詰まったときに引く場所

第3章には、第2章で使ったプロンプトも改めてまとめています。 録画後の依頼、最小版コード作成、.env、止まった状態の報告、途中再開、ログ保存。 参照しに行ったときは、第3章だけ見ればだいたい拾えるようにします。

A-1 ボタンを押しても何も起きない A-2 フォームに入力しても反映されない A-3 要素が見つからない B-1 ページ遷移を待てない C-1 iframeで操作できない D-3 途中から再開したい

dispatch問題

フォームに文字が入っているように見えるのに、保存すると未入力扱いになることがあります。 これは「入力欄が変わったよ」という合図がサイト側に伝わっていない状態です。 

フォームに入力したテキストが、保存時に反映されません。
ReactのControlled Componentによる問題だと思います。
dispatchEventを使った方式で入力するように修正してください。

FAQ

よくある質問

CodexとClaude Code、どちらがいいですか?

基本的にはどちらでも始められます。ただし教材執筆時点、2026年5月時点では、 個人的にはCodexが一歩リードしていると感じています。今から始める非エンジニアの人には、まずCodexがおすすめです。

月3,000円の課金は必要ですか?

本気で自動化ツールを作るなら、有料プラン前提のほうが進めやすいです。 ただし、APIキー、外部サービス連携、エージェント機能を使う場合は月額とは別に従量課金が発生することがあります。 料金画面、利用上限、予算上限を確認し、小さな作業から試してください。

利用規約違反になりませんか?

サービスごとに規約が違います。自分のアカウント内の作業でも、 自動アクセス、bot、データ取得、大量送信、投稿やDM送信の自動化が禁止されていればNGになり得ます。 他社のお問い合わせフォームへ営業文面を自動送信する使い方は推奨しません。 まずは自分の少量の定型作業から始め、公式APIや公式連携がある場合はそちらを優先してください。

CAPTCHAが出るサイトも自動化できますか?

CAPTCHAは「私はロボットではありません」や画像選択の認証です。 自動突破しようとせず、ログインとCAPTCHAは人間が手動で対応してください。 その後のアップロード、入力、保存など、CAPTCHAが出ない範囲の定型作業を自動化する分担が現実的です。

Windowsでもできますか?

Windowsでも可能なケースはありますが、この教材はMacでの実例が中心です。 環境構築、録画方法、ターミナル操作、ファイルパス、ブラウザ起動方法が変わります。 Windowsユーザーは、まずCodexかClaude Codeに「Windows向けに手順を書き換えて」と頼んでください。

パスワードやAPIキーはどう扱いますか?

秘密情報は.envに入れ、Codex、Claude Code、ChatGPTのチャット欄には貼りません。 GitHubに上げないよう.gitignoreに.envを入れてください。 誤って公開した場合は、そのパスワードやAPIキーを変更してください。

サイトのデザインが変わったら壊れますか?

壊れることがあります。軽い修正なら短時間で済む場合もありますが、 初心者の場合は数時間から数日かかることもあります。 エラー文、スクショ、どこで止まったかをCodexかClaude Codeに渡すと、直すための材料になります。

どうしても詰まった場合は相談できますか?

購入者向け公式LINEで相談できます。 公式LINE: https://lin.ee/wzvCa0v 購入者が増えた場合はDiscordでのアフターサポートも検討しています。 ただし個別環境すべてを完全に代行・保証するものではありません。 エラー文、スクショ、OS、どこまでできたかを添えて相談してください。
